Ubuntu 20.04 上部署 Docker +Deepseek + VLLM + RAGFlow 的详细指南

下面为你提供 Docker +Deepseek + VLLM + RAGFlow 的安装部署指南。
环境准备

  • 硬件：建议使用 GPU 服务器（至少 16GB 显存，如 RTX 3080）
  • 系统：Ubuntu 20.04/22.04 LTS
  • 软件：Docker 20.10+、Docker Compose v2、NVIDIA Container Toolkit
    安装步骤

1. 安装 Docker 和 Docker Compose

2. 安装 NVIDIA Container Toolkit

3. 部署 vLLM 服务

4. 部署 RAGFlow

配置与验证

    1. 验证服务状态
    1. 访问 RAGFlow 界面

打开浏览器访问 http://localhost:3000，按照界面指引完成初始配置：

  • 连接 vLLM API（地址为 http://vllm:8000）

  • 上传文档或添加数据源

  • 配置 embedding 模型和 RAG 参数

  • 测试 RAG 流程

    1. 常见问题解决
      GPU 显存不足：修改 docker-compose.yml 中的 QUANTIZATION 参数（如使用 q8_0 或 q4_0）
      模型下载缓慢：添加 Hugging Face 镜像源（修改 vLLM 服务的环境变量）

RAGFlow 连接失败：检查容器网络连通性，确保 vLLM 服务正常响应

扩展配置

    1. 持久化数据
    1. 自定义 Deepseek 模型
      修改 docker-compose.yml 中的 MODEL 环境变量，选择合适的 Deepseek 模型：

deepseek-ai/deepseek-llm-7b-base
deepseek-ai/deepseek-coder-6.7b-base
deepseek-ai/deepseek-llm-67b-base
后续步骤

    1. 配置 RAG 流程：在 RAGFlow 界面创建文档索引，设置检索参数
    1. 优化性能：根据实际需求调整模型量化级别和分块策略
    1. 集成应用：通过 RAGFlow 提供的 API 将服务集成到业务系统中

如果在安装过程中遇到问题，可以提供具体错误信息以便进一步排查。

hexo-tag-abcjs

Helps you to render standard music notation with abcjs on your blog posts

Install

Configuration
You can configure this plugin in _config.yml.

  • js - The abcjs library url. default: /js/abcjs_midi_5.9.1-min.js
  • css - The abcjs style url. default: /css/abcjs-midi.css
  • midi - Whether enable midi audio. default: true
  • animation - Whether enable absjs animation. When true, highlights the current notation. default: true
  • animationColors - The animated notation color. default: [‘#000000’, ‘#3d9afc’]
  • options - The abcjs Configuration. default as above
  • scriptId - The id of inserted script tag. default: hexo-tag-abcjs

Usage

  • autoplay - Enable autoplay
  • no-midi - Disabled midi audio
  • no-animation - Disabled audio animation
