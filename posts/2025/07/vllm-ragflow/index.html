  • 已安装 NVIDIA 驱动（版本 ≥ 525，支持 CUDA 12.0+）
  • 系统预留 ≥ 50GB 存储空间（用于模型缓存和数据存储）

5. 配置 RAGFlow 与使用

  1. 访问界面：浏览器打开 http://服务器IP（注册账号密码）

  2. 初始化配置：

  • 在「系统设置」→「LLM 配置」中，确认 vLLM 地址为 http://vllm-deepseek:8000/v1
  • 上传知识库文档（支持 PDF/Word/Markdown 等格式）
  • 模型类型：chat
  • 模型名称：deepseek-ai/deepseek-coder-1.3b-instruct
  • 基础url：http://vllm-deepseek:8000/v1
  • 最大token数：4096
  1. 测试 RAG 流程：
  • 在问答界面输入与文档相关的问题
  • 验证回答是否包含文档引用（确认 RAG 生效）

常见问题解决

  1. 模型下载失败：
  • 检查 HF_ENDPOINT 是否配置正确（国内推荐 https://hf-mirror.com）
  • 手动下载模型后挂载到 model_cache 卷（路径：/root/.cache/huggingface/hub）
  1. GPU 显存不足：
  • 改用更高量化级别：QUANTIZATION=q2_k（最低显存需求～6GB）
  • 切换更小模型：deepseek-ai/deepseek-llm-1.3b-chat（显存占用～4GB）
  1. RAGFlow 无法连接 vLLM：
  • 检查容器网络：docker network inspect llm-deepseek_default
  • 重启 vLLM 服务：docker restart vllm-deepseek

扩展建议

  • 高可用配置：添加 Nginx 反向代理与 SSL 证书
  • 性能优化：
    • 增加 GPU_MEMORY_UTILIZATION=0.9（利用 90% GPU 显存）
    • 启用模型预热：PRELOAD_MODELS=[{“name”: “deepseek-ai/deepseek-llm-7b-chat”, “parameters”: {“quantization”: “q4_k_m”}}]
  • 监控集成：添加 Prometheus + Grafana 监控 GPU 使用率与服务响应时间

通过以上优化，部署流程更健壮，同时兼顾了国内网络环境与资源限制，适合快速搭建企业级 RAG 问答系统。
