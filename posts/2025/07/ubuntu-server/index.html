安装 Ubuntu 22.04 Server 搭建服务器

以下是详细的安装流程，涵盖从准备工作到基础配置的全流程，适合新手参考

一、安装前准备

在开始安装前，需完成以下准备工作，避免安装过程中出现问题：

  1. 硬件要求
    Ubuntu 22.04 Server 对硬件要求较低，基础配置如下（根据实际用途可提升）：
  • CPU：至少 1 GHz 处理器（推荐双核及以上）
  • 内存：至少 1 GB RAM（推荐 2 GB 及以上，运行服务更流畅）
  • 硬盘：至少 20 GB 可用空间（机械硬盘 HDD 或固态硬盘 SSD 均可，SSD 速度更快）
  • 其他：网络接口（用于联网）、USB 接口（用于插入安装介质）
  1. 下载安装镜像

  • 访问 Ubuntu 官方下载页：Ubuntu 22.04 LTS Server

  • 选择 “ubuntu-22.04.x-live-server-amd64.iso”（x 为具体版本号，如 22.04.5），下载到本地。

  1. 制作启动介质（U 盘）
    需要将下载的 ISO 镜像写入 U 盘，作为安装启动盘：
  • 工具选择：
    • Windows 系统：使用 Rufus（简单易用，支持 UEFI/BIOS）
    • macOS/Linux 系统：使用 dd 命令（终端操作，如 sudo dd if=镜像路径 of=/dev/sdX bs=4M，注意替换 sdX 为 U 盘设备名）
  • 操作注意：写入前备份 U 盘数据（会被格式化），选择 “GPT” 分区格式（支持 UEFI 启动，更主流）。
  1. 服务器启动设置
    将制作好的 U 盘插入服务器，重启服务器并进入 BIOS/UEFI 设置：
  • 进入方式：开机时按提示按键（如 Del、F2、F12，不同品牌主板不同）
  • 关键设置：
    • 启动顺序：将 U 盘设为第一启动项（优先从 U 盘启动）
    • 关闭 “Secure Boot”（部分服务器可能需要，避免阻止非认证系统启动）
  • 保存设置并重启，服务器会从 U 盘启动，进入 Ubuntu 安装界面。

二、开始安装（图形化引导流程）

Ubuntu 22.04 Server 提供图形化安装界面，操作直观，按以下步骤逐步完成：

  1. 初始界面选择
    启动后首先看到选择界面：
  • 选择 “Install Ubuntu Server”（安装 Ubuntu 服务器版，而非 “Try” 试用），按回车进入。
  1. 语言与键盘设置
  • 语言选择：默认 “English”（推荐保持英文，避免后续命令行乱码，服务器一般用英文环境）。
  • 键盘布局：选择 “English (US)”（默认即可，如需其他布局可通过 “Detect Keyboard Layout” 检测）。
  1. 网络配置（核心步骤）
    服务器必须联网（后续需下载更新、安装软件），此步骤需配置网络：
  • 默认情况：如果服务器连接了有线网络且路由器支持 DHCP，系统会自动获取 IP 地址（显示 “Connected”），直接按 “Done” 进入下一步即可。
  • 手动配置（如需固定 IP）：
    1. 选择当前网络接口（如 “enp0s3”），按回车进入设置。
    2. 选择 “Edit IPv4”，将 “Method” 从 “DHCP” 改为 “Manual”。
    3. 填写信息：
      • Subnet：子网（如 192.168.1.0/24）
      • Address：服务器固定 IP（如 192.168.1.100）
      • Gateway：网关（路由器 IP，如 192.168.1.1）
      • Nameservers：DNS 服务器（如 8.8.8.8 或公司 DNS）
    4. 填写完成后按 “Save” 保存，确认 “Connected” 后按 “Done”。
  1. 代理与镜像源设置
  • 代理服务器：如果服务器需通过代理联网，填写代理地址（如 http://代理IP:端口）；无代理则留空，按 “Done”。
  • 镜像源设置：默认使用 Ubuntu 官方源（http://archive.ubuntu.com），但国内用户可替换为 国内镜像源（如阿里云、华为云），加快后续软件下载速度：
    • 替换为阿里云源：http://mirrors.aliyun.com/
    • 操作：删除默认地址，输入新地址后按 “Done”。
  1. 磁盘分区（重要，需谨慎）
    此步骤分配硬盘空间，决定系统如何使用硬盘，推荐两种方式：

方式 1：自动分区（新手首选）

  • 选择 “Use an entire disk”（使用整个硬盘），系统会自动创建分区（根目录 /、swap 交换分区等），适合单硬盘、无特殊需求的场景。
  • 确认硬盘：选择需要安装系统的硬盘（注意不要选错，避免格式化其他硬盘）。
  • 勾选 “Set up this disk as an LVM group”（推荐，LVM 支持后续动态调整分区大小，灵活度高）。
  • 按 “Done” 进入确认界面，选择 “Continue”（提示会格式化硬盘，确认无误后继续）。

方式 2：手动分区（适合进阶需求）
如果需要自定义分区（如分离 /home、/var 目录，或多硬盘规划），选择 “Custom storage layout”，手动创建分区：

  • 基本分区建议（以 100GB 硬盘为例）：
    • /boot：512MB（启动分区，必须）
    • swap：2GB（交换分区，内存小于 4GB 时建议设置，等于或大于内存）
    • /（根目录）：剩余全部空间（存放系统和软件）
  • 操作：选择硬盘 -> “Add GPT Partition” -> 填写大小、挂载点、文件系统（默认 ext4）。
  1. 用户设置（创建管理员账户）
  • Profile setup：设置服务器的基本用户信息：
    • Your name：用户全名（如 “Server Admin”）
    • Server name：服务器主机名（如 “ubuntu-server”，后续可通过 hostname 命令修改）
    • Username：登录用户名（如 “admin”，避免用 “root” 直接登录）
    • Password：登录密码（需复杂，至少 8 位，包含大小写、数字和符号）
    • 勾选 “Install OpenSSH server”（必须！安装 SSH 服务，后续可通过远程工具如 Xshell、Putty 登录服务器）。
  • 按 “Done” 进入下一步。
  1. 可选软件安装（按需选择）
    此步骤可选择预装一些常用服务（通过 “Snaps” 安装，轻量化）：
  • 可选项目：如 “Docker”“Kubernetes”“LAMP Server” 等，新手暂时不选（后续需要时再手动安装更灵活）。
  • 直接按 “Done”，进入安装过程（系统开始写入硬盘、安装基础组件）。
  1. 完成安装
  • 安装过程需要几分钟（取决于硬件速度和网络，若之前选了镜像源，速度会更快）。
  • 出现 “Installation complete” 提示后，按 “Reboot Now” 重启服务器。
  • 重启前会提示 “Please remove the installation medium”，拔出 U 盘，按回车确认，服务器将从硬盘启动。

三、安装后验证与基础配置

重启后，服务器会显示登录界面（提示 “username@hostname:~$”），输入之前设置的用户名和密码登录，完成以下验证和配置：

  1. 验证登录与网络
  • 登录后，输入 ip addr 命令，查看网络是否正常（是否有 IP 地址）。
  • 输入 ping baidu.com，测试能否联网（能收到回复说明网络正常，按 Ctrl+C 停止）。
  1. 切换到 root 用户（可选）
    默认用户是普通用户，需使用 sudo 执行管理员命令。若习惯用 root，可启用 root 账户：
  1. 更新系统（推荐）
    安装完成后，先更新系统到最新版本，修复漏洞：
  1. 远程登录测试（关键）
    在本地电脑通过 SSH 工具（如 Xshell、FinalShell）连接服务器，验证远程管理是否正常：
  • 连接信息：IP 地址（通过 ip addr 查看）、用户名（安装时设置的）、密码。
  • 命令行连接（本地为 Linux/macOS）：ssh 用户名@服务器IP（如 ssh admin@192.168.1.100）。
  • 若能成功登录，说明 SSH 服务正常，后续可脱离服务器显示器，远程管理。

四、常见问题与解决

  1. 安装时无法联网：
  • 检查网线是否插好，路由器是否正常。
  • 若 DHCP 失败，手动配置 IP（参考步骤 3 的 “手动配置”）。
  1. 分区时提示 “无可用磁盘”：
  • 确认硬盘已正确识别（在分区界面查看是否有硬盘列表）。
  • 若硬盘之前有其他系统，可先删除原有分区（选择硬盘 -> “Delete”）。
  1. 重启后无法进入系统：
  • 检查 BIOS 启动顺序是否已改为硬盘（安装后需将硬盘设为第一启动项）。
  • 若提示 “grub rescue”，可能是分区错误，重新安装并正确分区。
  1. 远程 SSH 无法连接：
  • 确认安装时勾选了 “Install OpenSSH server”，若未勾选，登录服务器后手动安装：sudo apt install openssh-server。
  • 检查防火墙：默认 Ubuntu 防火墙不拦截 SSH，若开启需放行：sudo ufw allow ssh。
